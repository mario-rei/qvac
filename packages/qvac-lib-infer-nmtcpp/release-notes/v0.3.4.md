# QVAC NMT Addon v0.3.4 Release Notes

This release introduces automated GitHub releases with mandatory release notes enforcement, improves documentation, and fixes CI workflow issues for mobile E2E testing.

## Features

### Automated GitHub Releases

The release process is now fully automated with enforced release notes. When a version bump is detected on merge to main, a GitHub release is automatically created using the corresponding release notes file from the `release-notes/` directory. This ensures every release has proper documentation and reduces manual release overhead.

A new `release-notes-check.yaml` workflow validates that release notes exist before allowing merges, and the `create-github-release.yaml` workflow handles the actual release creation.

### PR List Generation in Release Notes

The release notes command now automatically includes a "Pull Requests" section with links to all PRs included in the release, making it easier to trace changes back to their source.

## Bug Fixes

### Mobile E2E Test Workflow Fix

Fixed an issue where mobile E2E tests would fail when the "On PR Trigger" workflow was manually run via `workflow_dispatch`. The workflow now correctly handles this trigger type.

### Squash Merge Support for Release Changelog

The release changelog generation now properly supports squash merges, ensuring commit history is accurately reflected in release notes regardless of the merge strategy used.

## Documentation

### README Improvements

The README has been significantly expanded with improved documentation, clearer examples, and better organization to help users get started more quickly.
